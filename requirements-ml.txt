# StoryWeave — HEAVY NLP / ML dependencies (.venv-ml, Python 3.12)
# ---------------------------------------------------------------------------
# Why a separate venv: Python 3.14 (the .venv interpreter) lacks wheels for
# torch/GLiNER. .venv-ml pins Python 3.12 where these wheels exist. ML imports in
# the package are LAZY so the light .venv still imports cleanly; ML tests use
# importorskip and are skipped on the light gate.
#
# Setup (Windows):
#   "<python3.12>" -m venv .venv-ml
#   .venv-ml\Scripts\python -m pip install -U pip
#   .venv-ml\Scripts\python -m pip install -e .
#   .venv-ml\Scripts\python -m pip install torch --index-url https://download.pytorch.org/whl/cpu
#   .venv-ml\Scripts\python -m pip install -r requirements-ml.txt
#
# MODEL CACHE — MUST live on the project drive (F:), never C: (CLAUDE.md §3).
# The app sets this automatically (storyweave.config.Settings.hf_home ->
# <repo>/.hf-cache, applied via storyweave.nlp.extractor.configure_hf_cache before
# any HuggingFace import). For manual/ad-hoc runs in a shell, set it yourself:
#   set    HF_HOME=F:\Dev\...\storyweave\.hf-cache            (cmd)
#   $env:HF_HOME = "F:\Dev\...\storyweave\.hf-cache"          (PowerShell)
#   export HF_HOME="F:/Dev/.../storyweave/.hf-cache"          (bash)
# On Windows also keep HF_HUB_DISABLE_SYMLINKS=1 (avoids WinError 1314 without
# Developer Mode/admin). The .hf-cache dir is gitignored.
#
# Hardware target: GTX 1650 (4 GB VRAM) with CPU + 16 GB RAM fallback. The Phase-2
# gate ran on CPU; the small model also fits the GPU (gliner_device="cuda").
