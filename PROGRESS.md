## Done (sub-phase — qwen2.5:7b JSON-object `relation` bug hardened)
Robustness-only parsing fix (not a correctness/over-merge change). **NO schema change; citation gate / smallest-k reveal rule / `fence.py` untouched; `llm_enabled=False` default unchanged; Tier-1/2 untouched; no new SQL.**
- **The bug (now RESOLVED):** `qwen2.5:7b` intermittently emitted `relation` as a JSON **object** (not a string); the old `normalize_relation` did `raw.strip()` and **raised**, and `infer_identities`' broad `except` escalated that into marking the **WHOLE run `degraded`** — one malformed model output zeroed the entire identity pass.
- **The fix:** `normalize_relation(raw: object)` now **tolerates non-str without raising** — it coerces an unambiguous string out of a 1-element list or a single-value/known-key dict (`relation`/`type`/`value`/`name`/`label`), and otherwise returns `None` (the same "unrecognized → no edge" path as an unknown string). The model wrapper flags `IdentityVerdict.malformed_relation` when a non-null, non-str blob fails to coerce; the orchestrator **counts it (`malformed_relations`) and skips that pair-k only**, never the run. One bad relation now degrades **at most that one candidate pair** — the other pairs still write their edges.
- **Two DISTINCT, separately-tested paths (the whole point):** (1) **malformed pair** → skip + count, run continues (`degraded=False`); (2) **genuine model/infra failure** (model raises / can't connect) → still whole-run `degraded`, floor intact, exactly as before (`test_graceful_degradation_keeps_the_floor`, unchanged).
- **New counter:** `IdentityReport.malformed_relations` — observably distinct from a clean NO (`citations_rejected==0, malformed_relations==0`) and from `degraded`.
- **Tests (+2, deterministic, light `.venv`) — `tests/test_identity.py` (section E):** (a) one pair's verdict is malformed at every k while another pair is well-formed → the well-formed pair STILL writes its `ALIAS` edge at gold k=3, run NOT degraded, `malformed_relations≥1`; (b) `normalize_relation` unit: dict/list/number/bool inputs coerce-or-`None` WITHOUT raising, valid strings normalize as before. **Gate:** `pytest` **114 passed, 6 skipped** (was 112 → +2); P0 `test_fence.py`, the A/B regressions, the layered-reveal, the section-D SAME_AS/REINCARNATION tests, and the booby-trapped-`urlopen` no-socket guard all unchanged & green; `ruff` + `mypy` (strict, 57 files) clean.
